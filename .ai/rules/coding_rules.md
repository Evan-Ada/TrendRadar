# 编码规范（长文）

## 通用

- 修改范围遵循 Agent 边界：后端不动前端目录，前端不动后端目录。
- 单一职责；避免复制粘贴大块逻辑，优先抽取共用函数（放在合适的 `utils/` 或 `composables/`）。
- 命名：语义清晰；后端 camelCase 与现有文件一致；前端遵循 Vue 项目惯例。

## 后端（sensoube）

- 路由使用已有异步包装与校验模式；错误交给统一错误处理中间件。
- 禁止 `console.log` 泄露 token、银行卡、身份证等；生产路径用 logger。

## 前端（senso_admin）

- API 调用走项目统一的 HTTP 封装（若有）；错误提示对用户可读。
- 大型表单注意校验与加载态。
